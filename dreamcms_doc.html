DreamCMS 系统安装教程

DreamCMS 采用业界主流的 PHP + MySQL 架构开发，安装过程极其简单，仅需 3 分钟即可完成部署。以下是详细的安装步骤：

一、运行环境要求
  • 操作系统：Linux / Unix / Windows Server
  • Web 服务器：Apache / Nginx / IIS 6.0+
  • PHP 版本：PHP 5.2.x - PHP 5.4.x（推荐 PHP 5.3）
  • MySQL 版本：MySQL 5.0.x - MySQL 5.6.x
二、安装步骤
  1. 下载并解压：从 DreamCMS 官网下载最新的安装包，并解压到本地。
  2. 上传文件：使用 FTP 工具（如 FlashFXP）将解压后的所有文件上传至您的网站根目录。
  3. 设置文件权限：若您使用的是 Linux 服务器，请确保以下目录及其子目录具有可写权限（777）：
    /data/ /uploads/ /html/ /install/
  4. 运行安装向导：在浏览器中输入您的网站域名，系统会自动跳转至安装界面。若未自动跳转，请手动访问 http://您的域名/install/index.php。
  5. 填写配置信息：按照页面提示，同意授权协议，检测环境通过后，填写您的 MySQL 数据库连接信息（主机地址、数据库名、用户名、密码）以及管理员初始账号和密码。
  6. 完成安装：点击“开始安装”，等待系统写入数据库表。安装完成后，请务必**删除目录下的 /install/ 文件夹**，以防被他人恶意重新安装。
常见问题解答 (FAQ)
Q1：安装完成后，访问首页显示空白怎么办？

A：这通常是由于模版缓存目录无写入权限导致的。请检查 /data/tplcache/ 目录是否具有 777 写入权限。如果是 Windows 系统，请确保 IIS 匿名用户或 Users 组对该文件夹具有写入权限。

Q2：如何开启伪静态 (URL Rewrite) 功能？

A：登录后台，进入“系统设置” -> “核心设置”，将“运行模式”修改为“伪静态模式”。然后根据您的 Web 服务器类型，将官方提供的 .htaccess (Apache) 或 nginx.conf (Nginx) 规则文件部署到网站根目录下。

Q3：后台登录验证码不显示，或者提示验证码错误？

A：请确保您的 PHP 环境已开启 GD 库支持。您可以通过查看 phpinfo() 确认 GD 库状态。如果已开启但仍不显示，请检查文件编码是否为带 BOM 的 UTF-8 格式，BOM 头会导致页面输出多余字符从而破坏验证码图片的生成。

常用模版标签手册

DreamCMS 拥有强大而直观的标签调用系统，无需任何 PHP 基础，即可轻松调用网站数据。标签格式统一为 {dream:标签名 参数='值'}...{/dream:标签名}。

1. 栏目导航标签 (channel)

用于调用网站的栏目分类导航，支持高亮当前栏目。

{dream:channel type='top' row='10'} <li><a href="[field:typelink/]">[field:typename/]</a></li> {/dream:channel}
2. 文档列表标签 (arclist)

用于在首页或列表页调用指定栏目的文章列表，支持按时间、点击量排序。

{dream:arclist typeid='1' row='8' titlelen='40' orderby='pubdate'} <li> <span class="date">[field:pubdate function="MyDate('Y-m-d',@me)"/]</span> <a href="[field:arclink/]">[field:title/]</a> </li> {/dream:arclist}
3. 全局配置标签 (global)

用于调用系统全局变量，如网站名称、备案号、统计代码等。

网站名称：{dream:global name='cfg_webname'/} 备案号：{dream:global name='cfg_beian'/}
常用 API 接口说明

DreamCMS v2.0 引入了轻量级的 RESTful API 接口，方便开发者进行二次开发，或将网站数据对接至移动端 App、微信小程序等第三方平台。

1. 获取文章列表 API

接口地址：/api/v1/get_articles.php
请求方式：GET
请求参数：

  • typeid (int)：栏目 ID，默认为 0（获取全站）
  • limit (int)：获取数量，默认 10
  • page (int)：当前页码，默认 1

返回格式：JSON

{ "code": 200, "msg": "success", "data": [ { "id": 105, "title": "DreamCMS v2.0 正式版发布", "pubdate": "1224057600", "click": 1420, "writer": "admin" } ] }
管理员界面与安全配置

保障后台管理界面的安全是网站防黑客、防篡改的重中之重。DreamCMS 提供了灵活的后台管理入口配置以及多重安全加固手段。

一、如何进入管理员界面

在本地静态环境下，请直接访问同目录下的 admin.html，该页面即为 DreamCMS 管理员界面登录入口。

为了安全，运维界面和管理员账户都可以从管理员界面登录，但是会进入独立的两个后台。

进入后台后，可在左侧菜单选择「官方内容管理」，对新闻公告、角色介绍等页面内容进行查看、编辑与发布。修改完成后点击保存，全站静态缓存将自动更新。

🔒 默认管理员凭证说明

系统在首次安装成功后，默认生成的管理员账号为站长或管理员的注册电子邮箱，系统初始默认密码为 dreamCMS。请在首次登录后台后，立即前往“系统设置” -> “管理员管理”中修改您的初始密码，以保障系统安全。

二、后台登录保护配置

DreamCMS 后台登录界面内置了以下安全防护特性：

  • 验证码机制：默认开启 GD 库图形验证码，防止机器人使用字典进行暴力破解。
  • 登录尝试限制：如果连续输入密码错误超过 5 次，系统将自动锁定该管理员账号及登录 IP，锁定时间为 30 分钟。
  • 操作日志审计：系统会自动记录每一位管理员的登录时间、登录 IP 以及执行的所有后台操作（如发布文章、修改配置、删除用户等），方便站长随时审计安全状况。
